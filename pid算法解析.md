# PID 算法解析

## 1.基本原理

* 1-> PID是简单的控制算法，其中P是比例环节，I是积分环节,D是微分环节

* 2-> 简单来说就是采用误差的控制来进行3个环节的调整来做出对目标控制

* 3-> 误差值（error） = target - actual ，通过获取error值进入以上pid的环节，来进行调整actual，此时就形成了一个闭环控制

* pid基本框图

  ```mermaid
  		graph LR;
      target -->|设定值| midvalue[+-];
      midvalue -->|误差=目标-实际| error --> kp*error;
      error -->|积分| ki*integration_error_dt;
      error -->|微分| kd*derror/dt;
      ki*integration_error_dt & kp*error & kd*derror/dt -->|PID输出| out_value;
      out_value -->|控制信号| control;
      control -->|系统响应| actual;
      actual -->|反馈| midvalue;
  ```

  

* 定义式:
  $$
  error(t) = target(t) - actual(t)
  $$

  $$
  outvalue = k_p*error(t)+k_i\int_{0}^{t}{error(t)}dt+k_d*(\frac{derror(t)}{dt})
  $$

### 1.2各项含义

* 1.比例项--P

  * **公式**:

  $$
  out(t) = k_p * error(t)
  $$

  ​	可以看出比例项的输出仅仅取决于当前时刻和其他时刻无任何关系,示例:k_p = 100;error=10;那么输出的值就是1000;如果kp项的值越大调节越快,但是超调也会增加,当仅仅使用纯比例项来控制系统,会存在静态误差,这里的静态误差通俗解释就是存在损耗,理论上解读后续补充

  * **稳态误差补充**

    

* 2.积分项--I

  * **公式**
    $$
    outvalue = k_p*error(t)+k_i\int_{0}^{t}{error(t)}dt
    $$
    积分项的输出值取决于0~t时刻所有时刻误差的积分,与历史时刻有关,在程序里其实就是累加,然后乘上积分系数ki,一般积分项不单独使用,因为调节很慢,会导致滞后产生严重,同时积分项可以消除稳态误差

    ki系数越大,稳态误差调节的也就越快.

  * 部分曲线图

    ```plaintext
    value
    ^
    |					  /\
    |	 target |--------/--\-------------------------------------| 1.0
    |	 actual |       /   \------------------------------------| 1.0
    |		    |	  /    									      |
    |	   	    |	 /	    	kp=2     ki=0.05				  |
    |		    |   /         								      | 
    |      	    |  /		  						 	          | 
    |	   	    |/									 			  |
    |-----------|-------------------------------------------------|--------->
    ```

    可以从上图看出引入了积分项后稳态误差消除了,实际值和给定值相同了

* 3.微分项--D

  * 公式
    $$
    outvalue = k_perror(t)+k_i\int_{0}^{t}{error(t)}dt+k_d(\frac{derror(t)}{dt})
    $$
    微分项在数学里是求导,也就是求斜率,反应的是当前输出值,当前时刻误差变化的斜率,如果误差突变那么此项的作用就展现了,就是增加这个系统的阻尼,阻止这个系统突然变化,有效的解决系统超调的问题,但是kd项权重过大会让其系统运行变卡顿.

## 2.PID形式变换



了解基本的形式后,对PID有了部分的调节,但是上述给出的PID公式都是连续形式,应用到单片机系统内不好操作,所以需要将连续形式来进行离散化处理.
		如何进行离散化，在单片机的控制下，我们只能使用周期性来进行调控，所以进行 离散化的思想就是在固定周期下对其进行解析。

下方是连续形式的PID
$$
outvalue = k_perror(t)+k_i\int_{0}^{t}{error(t)}dt+k_d(\frac{derror(t)}{dt})
$$
离散形式如下
$$
out(k) = K_p*error(k)+k_i*T\sum error(j) +k_d*(\frac{error(k)-error(k-1)}T)
$$

离散形式的每一项如何得来

比例环节:首先分析最简单的P环节:在这个环节我们只需要建立当前周期的误差进行系数缩放较为简单
积分环节:积分本来就是求不规则图形的面积大小(此项表示不准确),在数字电路中需要选定一个时间周期来进行调控,所以离散化需要增加一个调控周期乘以每一次的误差值,再来进行求和.

微分环节:某一点的微分就是求切线的斜率,dy/dx





## 3.位置式PID和增量式PID

位置式:
$$
out(k) = K_perror(k)+k_iT\sum error(j) +k_d*(\frac{error(k)-error(k-1)}T)
$$


增量式:
$$
\Delta out(k)=k_p*(error(k)-error(k-1))+k_i*error(k)+k_d(err(k)-2error(k-1)+error(k-2))
$$
可以看出这里的增量式很方便没有复杂的乘除运算,运算速度快,但是这是增量式,直接使用无法直接给被控对象,如果被控对象没有积分功能,将无法使用,但是只要将输出结果累加起来就行,也可以达到位置式的效果.也是输出的全量





## 4.代码实现思路

* 1.确定一个调控周期,1ms10ms....针对你的被控对象的速率,理论上越快越稳定,但是受硬件影响
* 2.一般建议放在中断里面做调控,如果不上操作系统的话.
* 3.直接定义变量kpkikd,然后代入上面的公式,进行计算pid输出out值,out值给到被控对象,看能否实现对其给定和实际值进行追踪.



## 5.PID单闭环

* PID单闭环,如最开始的基本PID一样,只有一个量来进行反馈,从而来达到调控的目的,最核心的地方是要做出设定正确的时间周期T,以来进行PID调控





## 6.PID调控的改进

* 这个改进环节主要是针对积分和微分环节,因为比例相很好调控,只和当前值有关

### 6.1 积分环节的改进

* 积分限幅-改进积分环节

* 在积分环节会出现积分饱和的现象,产生这个现象是因为:被控器件断电,被控器件堵转等,而主控器件依旧在运行,如果此时被控器件恢复正常,那么就会以最大的值运行,而且还无法通过给定制进行调控,非常危险,所以我们需要给积分环节给一个最大的现定值,依次来控制积分环节不过度饱和的情况.在软件中实现很简单.

  ```c
  /*方法1-对误差积分进行限幅*/
  if(A>LIMIT_VALUE_MAX) A=LIMIT_VALUE_MAX
  else if(A<LIMIT_VALUE_MIN)A=LIMIT_VALUE_MIN
  /*方法2-对整个积分项进行限幅*/
  Intout += k_i * Error;
  if(Intout>LIMIT_VALUE_MAX) Intout=LIMIT_VALUE_MAX
  else if(Intout<LIMIT_VALUE_MIN)Intout=LIMIT_VALUE_MIN
  
  ```



* 上下限值确定方法

  1.计算法:积分项占输出值的权重,例如输出最大-100-100,ki值0.2,那么上下限就是-500-500







* 积分分离-改进积分环节

* 众多周知积分环节其实是为了消除稳态误差来增加的环节,软件实现如下

  ```c
  if(fabs(Error0) < limt_value)
  {
      errorint+=Error0
  }
  else
  {
      errorint = 0;
  }
  ```
