# YMODEM协议

## 1-帧结构

* 1.1 Ymodem帧格式解析

  | 简写   | SOH/STX | unuse  | PN   | ~PN      | DATA     | CRC-16-H | CRC-16-H |
  | ------ | ------- | ------ | ---- | -------- | -------- | -------- | -------- |
  | 名称   | 帧头    | //0x00 | 包号 | 包号反码 | 数据域   | 校验高位 | 校验低位 |
  | 字节数 | 1       | 1      | 1    | 1        | 128/1024 | 1        | 1        |

  帧头含义:SOH代表启动帧,STX代表数据帧.SOH开头包总长度:128+3+2=133字节,STX开头包总长度:1024+3+2=1029

  包号定义:0~255,大于255归零计算

  包号反码:取反包号0x00==>0XFF

  CRC16校验:计算数据为数据域内的校验,不包含帧头包号反码

* 1.2 协议帧内符号解析

  | 符号 | 数值(hex) | 含义           |
  | ---- | --------- | -------------- |
  | SOH  | 0X01      | 128字节数据包  |
  | STX  | 0x02      | 1024字节数据包 |
  | EOT  | 0X04      | 传输结束       |
  | ACK  | 0X06      | 应答信号       |
  | NAK  | 0x15      | 非应答信号     |
  | CA   | 0X18      | 停止传输       |
  | C    | 0x43      | 请求数据       |

  

## 2-传输示例

* 连接建立环节

  单片机打印菜单,上位机发送字符'1',单片机收到'1'后,传输'C'字符,此时代表连接建立成功,如果传输的非'C'或其他表示连接没能建立.

* 起始帧

  上位机发送以SOH开头的帧-帧示例如下:

  | 帧头 | 未使用 | 包号 | 包反码 | 文件名        | 文件大小        | 填充区           | 校验高位 | 校验低位 |
  | ---- | ------ | ---- | ------ | ------------- | --------------- | ---------------- | -------- | -------- |
  | SOH  | 0X00   | 0X00 | 0XFF   | FILENAME+0X00 | FILE SIZE +0X00 | 未使用到的(0x00) | CRC-H    | CRC-L    |

  其中包号固定0,filename是文件名称,文件名称后必须加0x00作为结束符号,filesize文件大小也必须加0x00作为结束符,剩于填充区补齐0x00.

  示例：文件名为ALL_DEMO.bin,文件大小：33.8k（34656字节），那么发送端发送的数据如下
  
  0x01 0x00 0x00 0xff  // 
  // 0x41 0x4C 0x4C  0x5F  0x44 0x45 0x4D 0x4F 0x2E 0x62 0x69 0x6E 0x00 // 13 长度
  // 0x33 0x34 0x36 0x35 0x36 0x00 //6 长度
  0x00 // 128-13-6=109 长度
  83 3C //CRC
  接收端：回复（正确）ACK

* 数据帧

  | 帧头 | 包号 | 包反码 | FLASH数据1024字节 | 校验高位 | 校验低位 |
  | ---- | ---- | ------ | ----------------- | -------- | -------- |
  | STX  | 0X01 | 0XFE   | flash数据         | CRC_H    | CRC-L    |

​	传输示例:

​	文件里面的内容: D8 12 00 20 D9 38 00 08 .........1k

​    0x02 0x01 0xfe D8 12 00 20 D9 38 00 08 ............33 AC 

​	依次读取文件内的内容,然后发送给接收端,接收端验证正确后会回复ACK,错误则会回复NAK,发送端重传本帧即可

​	注意:在传输过程中,如果内容未超过文件中的内容未超过128字节则采用SOH帧

​	超过128字节则一直采用STX帧

* 结束文件传输

  发送端发完数据后,发送结束帧*2

  | 帧头 | 未使用 |
  | ---- | ------ |
  | EOT  | 0X00   |

* 结束帧

  发送SOH帧

  SOH 0X00 0XFF 0X00*128 CRC-H CRC-L
