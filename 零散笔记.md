### 1.1 Program Size

* 示例:Program Size: Code=14560 RO-data=312 RW-data=20 ZI-data=3332  

  flash占用初步估计:flash_size =  Code + RO-data = 14872(字节)

  ram占用初步估计:ram_size =  RW-data=20 + ZI-data=3332  = 3352 (字节) 

  ram是初步估计,具体还是要看map文件.下面这个更加的具体

  ~~~text
  
        Code (inc. data)   RO Data    RW Data    ZI Data      Debug   
  
       14560        742        312         20       3332     582594   Grand Totals
       14560        742        312         20       3332     582594   ELF Image Totals
       14560        742        312         20          0          0   ROM Totals
  
  ==============================================================================
  
      Total RO  Size (Code + RO Data)                14872 (  14.52kB)	 // flash (RO)
      Total RW  Size (RW Data + ZI Data)              3352 (   3.27kB)	// ram()
      Total ROM Size (Code + RO Data + RW Data)      14892 (  14.54kB) 
  
  ==============================================================================
  
  ~~~

### 1.3 keil编译时间过长解决方法

* Windows下一般是Windows的病毒管理出问题,解决方式如下

  ~~~c
  1.WIN+R
  2.输入(msconfig)->回车确定
  3.点击服务->点击隐藏Microsoft
  4.把Microsoft PC Mannager Service 前面的小勾取消
  5.点击确定应用保存,然后重启.
  6.特别检查一下keil的小扳手编辑线程数->ohter->Parallel->Number of paraller jobs->选择最大
  7.如果以上无用,直接禁用全部然后在一个一个解除(一般都是这个PC Mannager)
  ~~~

### 1.4 使用AC6编译器,创建rtos工程报错

~~~c
1.找到CUBEMX-存储固件库的地方-一般是:C:\Users\lshml\STM32Cube\Repository
2.找到对应的固件库 \STM32Cube_FW_H7_V1.12.1\Middlewares\Third_Party
3.Middlewares\Third_Party\FreeRTOS\Source\portable
4.主要是这个里面的有很多的宏定义是内联或者其他的方式,AC6编译会有问题.
5.GCC的文件夹直接复制,改名为->RVDS,之前的RVDS->RVDS-OLD
6.重新使用CUBMX生成工程重新编译即可.
~~~

## 1.6 JLINK添加其他厂家的dev

~~~text
1.windows 按下WIN+R 
2.输入框内输入%APPDATA%\SEGGER
3.回车进入
4.在当前目录下新建文件夹-JLinkDevices
5.JLinkDevices在这个文件夹下新建文件夹Devices，同时在JLinkDevices 把厂家的.XML文件放入
6.把厂家的FLM名字按照他的格式打包放入
7.重新打开jlink.exe
8.选择对应设备进行hex文件烧录
~~~

## 3.STM32HAL库一些知识点

### 3.1 文件名：stm32h7xx_hal.c

这个文件比较杂，像基准电压大小配置，EXTI配置，IO补偿配置等都在这个文件里面设置

*   函数：HAL_Init 

    设置NVIC-分组为4

    设置滴答定时器的时间基准为1ms中断一次

*   函数:HAL_Deinit

    此函数用于复位HAL库和滴答时钟，复位了AHB1,2,3,4的时钟以及APB1L，APB1H，APB2,3,4的时钟。

*   IO补偿的函数也有需要自己去看

### 3.2文件名：stm32h7xx_hal_rcc.c 

这个文件主要是先的是内外部时钟和总线时钟的配置：HSE\HSI\LSE\CSI\LSI->APB123\SYSTICK......相关时钟的配置。比较庞杂
