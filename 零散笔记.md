## 1.stm32相关

### 1.1 Program Size

* 示例:Program Size: Code=14560 RO-data=312 RW-data=20 ZI-data=3332  

  flash占用初步估计:flash_size =  Code + RO-data = 14872(字节)

  ram占用初步估计:ram_size =  RW-data=20 + ZI-data=3332  = 3352 (字节) 

  ram是初步估计,具体还是要看map文件.下面这个更加的具体

  ~~~text
  
        Code (inc. data)   RO Data    RW Data    ZI Data      Debug   
  
       14560        742        312         20       3332     582594   Grand Totals
       14560        742        312         20       3332     582594   ELF Image Totals
       14560        742        312         20          0          0   ROM Totals
  
  ==============================================================================
  
      Total RO  Size (Code + RO Data)                14872 (  14.52kB)	 // flash (RO)
      Total RW  Size (RW Data + ZI Data)              3352 (   3.27kB)	// ram()
      Total ROM Size (Code + RO Data + RW Data)      14892 (  14.54kB) 
  
  ==============================================================================
  
  ~~~

  

###  1.2 BIN文件的生成方式

* 在keil里面可以点击魔术棒里面的user页面的After/build指令下添加,文件生成的默认路径是在:mdk-arm下

  ~~~text
  fromelf --bin --output=project_name.bin !L   
  //project_nam 这个是生成的名字,可以自定义project_name.bin
  ~~~

  

### 1.3 keil编译时间过长解决方法

* Windows下一般是Windows的病毒管理出问题,解决方式如下

  ~~~c
  1.WIN+R
  2.输入(msconfig)->回车确定
  3.点击服务->点击隐藏Microsoft
  4.把Microsoft PC Mannager Service 前面的小勾取消
  5.点击确定应用保存,然后重启.
  6.特别检查一下keil的小扳手编辑线程数->ohter->Parallel->Number of paraller jobs->选择最大
  7.如果以上无用,直接禁用全部然后在一个一个解除(一般都是这个PC Mannager)
  ~~~

### 1.4 keil软件图形化配置-使用宏定义

* configuration wizard-主要是用的是这个页面功能

  主要是下面的<<<>>>里面的内容配置格式,必须按照这么写目前主要使用这些,给了部分示例.具体参考keil的使用文档,具体使用网址和教程如下面的链接

  [µVision User's Guide](https://developer.arm.com/documentation/101407/0543/Utilities/Configuration-Wizard?lang=en)

  ~~~c
  // <<< Use Configuration Wizard in Context Menu >>>  //这是主要
  #ifndef __CONFIG_H
  #define __CONFIG_H
  // 宏定义使能
  // <h>Debug_Configuration     //标题                          
  
  // <c1> debug_mode_label	//次级显示内容
  // <i> config debug mode ,defualt : disable //<i>描述符
  //#define enable					//自己的宏定义
  // </c>
  
  // </h> 	//结束
  // 配置字符串在左边
  // <h>Soft_Version_Configuration
  //  <s.20>Soft_Version String
  //  <i>config Soft Version info lenth max:20
  #define Soft_Version              L"v1.0.0"
  // </h> 
  
  // 配置选择框
  // <h>SSD13XX_Configuration                               
  //   <o>SSD13XX_Config_Screen_size
  //                        <1=> 0.96 
  //                        <2=> 1.3 
  //   <i> Defines priority for Timer Thread
  //   <i> Default: 0.96 
  #ifndef SCRENN_SIZE 
   #define SCRENN_SIZE   1
  #endif
  //	 数字输入框
  //   <o> Default Device row size <0-144>
  //   <i> Defines default Device row  = 128
  //   <i> Default: 128
  #ifndef DEV_ROW
   #define DEV_ROW     128
  #endif
  //	 数字输入框
  //   <o> Default Device Col Size <0-16>
  //   <i> Defines default Device row  = 8
  //   <i> Default: 8
  #ifndef DEV_COL
   #define DEV_COL     8
  #endif
  // </h> 
  
  #endif //__BEARPI_CONFIG_H
  // <<< end of configuration section >>>  //这是主要
  ~~~

## 3.STM32HAL库一些知识点

### 3.1 文件名：stm32h7xx_hal.c

这个文件比较杂，像基准电压大小配置，EXTI配置，IO补偿配置等都在这个文件里面设置

*   函数：HAL_Init 

    设置NVIC-分组为4

    设置滴答定时器的时间基准为1ms中断一次

*   函数:HAL_Deinit

    此函数用于复位HAL库和滴答时钟，复位了AHB1,2,3,4的时钟以及APB1L，APB1H，APB2,3,4的时钟。

*   IO补偿的函数也有需要自己去看

### 3.2文件名：stm32h7xx_hal_rcc.c 

这个文件主要是先的是内外部时钟和总线时钟的配置：HSE\HSI\LSE\CSI\LSI->APB123\SYSTICK......相关时钟的配置。比较庞杂
